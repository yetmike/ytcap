# Security Policy

## Reporting a vulnerability

If you believe you have found a security issue in ytcap, please **do not** open a public GitHub issue.

Instead, use GitHub's [private vulnerability reporting](https://github.com/yetmike/ytcap/security/advisories/new) so we can discuss and fix the issue before it becomes public.

Please include:

- A description of the issue and its impact
- Steps to reproduce, or a proof-of-concept
- The version of ytcap affected (`ytcap version`)

We aim to acknowledge reports within 7 days and ship a fix for confirmed issues in a timely manner.

## Supported versions

Only the latest release receives security fixes.
